======= ОБЩИЙ РАЗДЕЛ =======

1. Вид приложения на релизе

    Набор файлов с кодом на языке Python и один или несколько конфигурационных файлов с данными приложения.

2. Структура файлов приложения

    Модуль верхнего уровня, являющийся точкой входа в приложение. В этом модуле осуществляется импорт дополнительных модулей, инициализация приложения и управление основными этапами работы  приложения.

    Дополнительные модули, содержащие импорты модулей/пакетов из стандартной библиотеки. В основном состоят из функций, обеспечивающих внутреннюю логику работы приложения.

    Текстовый(-е) файл(-ы) для хранения данных приложения.

3. Структуры хранения данных

    # help.txt

        Текст с описанием приложения и описанием игры.

    # players.ini

        [Player1]
        wins = 2
        fails = 3
        ties = 5

        [Player2]
        wins = 0
        fails = 0
        ties = 0

    # saves.ini

        [PlayerA;PlayerB]
        turns = 3,7,4,9,5

        [#2;PlayerB]
        turns = 5,7,1

4. Работа с данными в приложении

    # структура данных для работы с именами и статистикой игроков

        STATS = {'Player1': {'wins': 2, 'fails': 3, 'ties': 5, 'training': False},
                 'Player2': {'wins': 0, 'fails': 0, 'ties': 0, 'training': True}, }

    # структура данных для работы с сохранёнными партиями

        SAVES = {('PlayerA', 'PlayerB'): [3, 7, 4, 9, 5], }

    # структуры данных для работы с ходами текущей партии

        TURNS = [5, 3, 9, 2, 1]
        BOARD = ['X', 'O', 'O', ' ', 'X', ' ', ' ', ' ', 'X']


======= ЭТАПЫ РАБОТЫ =======

1. Чтение данных из текстовых файлов данных

2. ЕСЛИ первый запуск:
        вывод раздела помощи

3. Запрос имени игрока
    а) ЕСЛИ имени нет в базе игроков:
            добавление запись об игроке в базу игроков
            обновление текстовых файлов данных
    б) добавление имени игрока к списку активных игроков

# суперцикл

    4. Ожидание ввода команды

    # new

        5. Запрос режима игры:

            ЕСЛИ один человек:
                 6. Запрос уровня сложности бота
                     а) добавление имени бота к списку активных игроков

            ЕСЛИ два человека:
                 7. Запрос имени второго игрока
                     а) ЕСЛИ имени нет в базе игроков:
                             добавление запись о втором игроке в базу игроков
                             обновление текстовых файлов данных
                     б) добавление имени второго игрока к списку активных игроков

        8. Запрос очерёдности ходов:
            ЕСЛИ первый игрок выбирает ходить вторым:
                 изменение списка активных игроков

        9. Проверка включенного режима обучения у активных игроков

        # партия

            10. ЕСЛИ обучение:
                     вывод игрового поля с пронумерованными клетками

            11. Цикл раунда:

                12. Запрос хода игрока
                     а) ЕСЛИ обучение:
                             вывод подсказки к вводу хода
                     б) ЕСЛИ ввод пустой:
                             сохранение незавершённой партии
                        ЕСЛИ ввод не пустой:
                             ЕСЛИ ввод корректный:
                                  переход к этапу 15
                             ЕСЛИ ввод не корректный:
                                  повторение текущего этапа

                ИЛИ 13. ЕСЛИ обучение:
                             вывод пояснения о ходе бота

                    14. Расчёт хода бота

                15. Обновление глобальных переменных, выполнение автосохранения и обновление текстовых файлов данных

                16. ЕСЛИ обучение:
                         вывод подписи, чей ход на игровом поле

                17. Вывод игрового поля со сделанным ходом

                18. ЕСЛИ есть победная комбинация:
                         переход к этапу 19
                    ЕСЛИ нет победной комбинации:
                         ЕСЛИ есть ничья
                              переход к этапу 19
                         ЕСЛИ нет ничьей
                              переход к этапу 11

        19. Удаление автосохранения и обновление статистики

        20. Переход к этапу 4

21. Обработка завершения работы приложения
